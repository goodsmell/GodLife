
# 🌟 GodLife — 나의 하루를 시각화하는 작은 시작
</br>
GodLife는 기상시간·운동·투두·일기·메모 등</br>
나의 하루를 가볍고 직관적으로 기록하고 분석하는 갓생 트래킹 웹서비스입니다.</br>
우테코 프리코스 오픈미션을 통해 스스로의 생활을 바꾸고 싶은 마음에서 시작되었습니다.

</br>
</br>

# ✨ 프로젝트를 만든 이유
저는 평생 일찍 일어나는 데 실패해온 사람입니다.</br>
규칙적인 루틴을 지키는 사람들을 보면 부러웠지만 막상 제 삶에서는 실천되지 않았습니다.</br>
프리코스를 하면서 깨달았습니다.</br>
나의 문제는 알고리즘이 아니라, 잠과 생활 습관이었다는 사실을.</br>
</br>
그래서 이번 오픈미션에서는</br>
내가 가장 어려워하고 두려워하는 것을 정면으로 마주보기로 했습니다.</br>
</br>
그래서 무작정 매일 6시에 일어나고, 매일 뛰었습니다.</br>
생활 리듬이 바뀌자 정신적으로 안정되고, 미래를 생각할 여유도 생겼습니다.</br>
</br>
“나처럼 고민하는 사람들이 많을 텐데,</br>
이 변화를 기록하고 분석하도록 돕는 도구가 있다면 좋지 않을까?”</br>
</br>
그래서 직접 만들기로 했습니다.</br>
</br>
# 👥 누구를 위한 서비스인가?

### ✔ 나 자신
루틴을 기록하고 눈으로 확인하며 살아보고 싶은 사람.

### ✔ 하루 관리가 어려운 사람
아침 시간을 잡고 꾸준함을 삶에 들이고 싶은 사람.

### ✔ 복잡한 앱 대신 미니멀한 도구를 원하는 사람
기록과 시각화를 깔끔하게 하고 싶은 사람.

</br>

# 🧩 해결하고 싶은 문제

- 내가 언제 일어나는지 감으로만 알고 있고, 객관적으로 확인이 어렵다.
- 기록은 하지만 그 기록이 데이터로 연결되지 않는다.
- 루틴 관리 앱은 많지만 ‘나의 하루’를 한눈에 보여주는 서비스는 부족하다.
- 작은 루틴이라도 시작하고 유지하기 위해 낮은 진입장벽의 가벼운 도구가 필요하다.

</br>

# 🚀 주요 기능 (기능 구현 목록)

### 1. 목표 설정 (온보딩)

- [x] 사용자는 이름을 설정할 수 있다
- [x] 사용자는 목표 기상 시간을 설정할 수 있다.
- [x] 사용자는 목표 러닝 기준을 설정할 수 있다.

### 1. 오늘의 갓생 기록하기

- [x] 오늘 날짜를 확인할 수 있다.
- [x] 오늘 명언을 확인할 수 있다.
- [x] 사용자는 오늘의 기상 시간을 기록할 수 있다.
- [x] 사용자는 **러닝 기록(거리 또는 시간)**을 입력할 수 있다.
- [x] 사용자는 오늘의 목표 달성률을 수치(%)로 기록할 수 있다.
- [x] 사용자는 TODO 리스트를 추가/완료/삭제할 수 있다.
- [x] 사용자는 일기를 작성하고 저장할 수 있다.
- [x] 사용자는 메모를 작성하고 저장할 수 있다.
- [x] 모든 기록은 IndexedDB에 저장되며 브라우저를 종료해도 유지된다.

### 2. 기록 조회

- [x] 사용자는 날짜 선택 UI를 통해 과거의 갓생 기록을 조회할 수 있다.
- [x] 선택된 날짜의 기상시간, 러닝, 목표달성률, TODO, 일기, 메모가 화면에 표시된다.
- [x] 해당 날짜에 기록이 없을 경우 “기록 없음” 화면이 표시된다.

### 3. 분석

- [x] 사용자는 월 단위 분석 페이지를 통해 지난 한 달의 갓생 패턴을 확인할 수 있다.
      ** 분석 페이지는 다음 지표를 제공한다. **
- [x] 기상 시간 그래프
- [X] 러닝 그래프
- [X] 월간 TODO 완료율
- [X] 월간 일기 작성률
- [ ] 분석 데이터는 IndexedDB에 저장된 기록을 기반으로 계산한다.

### 4. 설정

- [x] 사용자는 목표 기상 시간을 설정할 수 있다.
- [x] 사용자는 목표 러닝 거리 또는 목표 러닝 시간을 설정할 수 있다.
- [x] 사용자는 러닝 기록 방식을 거리(km)/시간(min) 중 하나로 선택할 수 있다.
- [x] 설정 값은 IndexedDB에 저장되며 페이지 전환 및 새로고침에도 유지된다.

</br>

# 🧱 아키텍처 & 디렉토리 구조 (요약)
```bash
src/
 ├── components/     # UI 컴포넌트
 ├── pages/          # 라우트 페이지
 ├── hooks/          # custom hooks (useDayLog, useTodayGoalLog 등)
 ├── db/             # IndexedDB 스키마 & Repository
 ├── utils/          # time/date helpers
 ├── types/          # TypeScript types
 └── constants/
```
IndexedDB를 사용하는 이유는 다음과 같다:

- 서버 없이도 기록이 영구 저장됨
- 가볍고 빠르며 로컬 기반 앱에 적합함
- 데이터를 그래프 분석에 즉시 활용 가능

</br>

#  🛤 개발 과정에서 배운 점

이 프로젝트는 단순한 기능 구현 이상의 의미가 있었다.</br>
기획 → 디자인 → 개발 → 배포까지 혼자 처음부터 끝까지 만든 첫 프로젝트이기 때문이다.

🌱 1. 작은 루틴이 개발을 지탱한다 </br>
매일 6시에 일어났고 운동을 했고 그 다음 개발을 했다. </br>
루틴이 잡히자 개발 속도와 집중력이 전혀 달라졌다.
</br>
</br>
🔧 2. 상태 관리를 단순하게 유지하는 법 </br>
데이터 종류가 많아질수록 하나의 큰 상태보다 </br>
각 기능별 Custom Hook으로 분리해 관리하는 방식이 안정적이었다.
</br>
</br>
📊 3. 시각화는 생각보다 강력하다 </br>
기록을 단순히 저장하는 것이 아니라 </br>
그래프로 보여주는 것만으로도 스스로의 생활이 달라지는 것을 경험했다. </br>
</br>
</br>
💡 4. 배포와 버전 관리의 중요성 </br>
Vite 기반 빌드 구조를 잡고 GitHub Pages로 배포하며 </br>
클라이언트 사이드 라우팅 문제 등을 해결하는 경험을 했다. </br>

</br>
